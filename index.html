AI Harness

automatic-chainsaw

A shared "Open Brain" context layer my wife and I both use — one Supabase + pgvector database, accessible to any MCP-compatible AI client (Claude Desktop, Claude Code, ChatGPT, Cursor). Every thought is tagged by owner, so we can scope to "mine" or "ours" without crossing wires.

Python ingest workers pull from Gmail, Google Calendar, Google Drive, and GitHub so context accumulates passively, not just by manual capture. Built on Nate Jones' OB1 pattern and shaped around Anthropic's Managed Agents architecture — the AI clients are the brain, the MCP server is the hands, the database is the durable session.

AI Skill Library for Salesforce Operations

A personal library of ~30 composable Claude Code skills I've built for the day-to-day reality of running Salesforce in production: triaging tickets, auditing flow failures across orgs, drafting change requests, deploying releases, investigating MuleSoft integration errors, and rolling up timesheets. Each skill is a focused agent with its own tooling, prompts, and verification steps.

The interesting part isn't the individual skills — it's how they compose. triage → itsm-investigate → change-request → cr-deploy runs an entire incident lifecycle through coordinated agents, with human approval at every consequential step. It's how I'm proving out the "Salesforce engineer + AI" model on real work, not toy examples.

Flow Audit Scoreboard

Most teams find out about broken Salesforce Flows when a user complains. This audit gets ahead of it: a heat-check across 11 production orgs pulls Flow failure counts, ranks the hottest, then drills into FlowInterview.Error records to surface the actual root cause — not just the failing element.

Each finding becomes a prioritized P3 tech-debt ticket in JSM with full reproduction context, and the running totals get published to a Confluence scoreboard the team actually reads. Turns an invisible category of debt into something with an owner and a trend line.
